🔌 Cách kết nối trong Tableau

Bước A – Mở Data Source cho Star Schema (cho các sheet tính toán linh hoạt)

Mở Tableau → Connect → Text File → chọn fact_sales.csv
File fact_sales là trung tâm, load trước tiên.
Kéo thêm dim_customer.csv vào canvas (dùng Relationship)
fact_sales.customer_id ←→ dim_customer.customer_id
Kéo thêm dim_product.csv
fact_sales.product_id ←→ dim_product.product_id
Kéo thêm dim_location.csv
fact_sales.location_id ←→ dim_location.location_id
Kéo thêm dim_time.csv
fact_sales.order_date ←→ dim_time.date (đổi kiểu cả hai về Date)
💡 Lưu ý Relationship vs Join
Tableau 2020.2+ dùng Relationship (mặc định) thay vì Join. Relationship thông minh hơn — không nhân bản row khi có 1-nhiều. Chỉ dùng Join khi cần tính toán trực tiếp trong cùng 1 bảng.

Bước B – Tạo Data Source riêng cho từng AGG file (tối ưu tốc độ)

Mỗi Dashboard nặng (RFM, Cohort, YoY) nên dùng Data Source riêng từ file AGG đã được pre-calculate, thay vì tính lại từ fact_sales.

Data → New Data Source → chọn file AGG tương ứng
Ví dụ: Dashboard 3 dùng agg_rfm_segments.csv + agg_cohort_retention.csv riêng biệt.
Đặt tên Data Source rõ ràng
"Star Schema", "RFM Data", "YoY Data", "Discount Data", "Shipping Data"

📐 Calculated Fields cần tạo (từ Star Schema)

Tên FieldCông thức TableauDùng cho
Total SalesSUM([Sales])Tất cả KPI
Total ProfitSUM([Profit])Tất cả KPI
Profit Margin %SUM([Profit]) / SUM([Sales])DB1, DB2
Loss OrdersCOUNTD(IF [Profit] < 0 THEN [Order Id] END)DB1, DB2
Loss Rate %[Loss Orders] / COUNTD([Order Id])DB2
Avg Order ValueSUM([Sales]) / COUNTD([Order Id])DB1, DB3
Shipping Cost RatioSUM([Shipping Cost]) / SUM([Sales])DB4
Is Loss MakerIF [Profit] < 0 THEN "Lỗ" ELSE "Lãi" ENDDB2
Discount BandIF [Discount]=0 THEN "0%" ELSEIF [Discount]<=0.1 THEN "0-10%" ELSEIF [Discount]<=0.3 THEN "10-30%" ELSEIF [Discount]<=0.5 THEN "30-50%" ELSE "50%+" ENDDB2

💡 Highlight Action vs Filter Action
Filter Action: Click → loại bỏ dữ liệu khác (chỉ còn cái được chọn)
Highlight Action: Click → làm nổi bật cái được chọn, mờ các cái còn lại (không mất data)
→ Nên dùng Highlight cho Scatter & Map để không mất context tổng thể.

🖥️ Dashboard Size & Layout

Cài đặt kích thước Dashboard:

⚠️ Lỗi thường gặp cần tránh
  • Không set Fixed Size cho Dashboard → layout vỡ khi mở trên màn hình khác
  • Dùng Join thay Relationship với 1-nhiều → bị nhân bản dòng, số sai
  • Không format số → KPI hiện 12627507 thay vì $12.6M
  • Màu không nhất quán → Furniture = cam ở DB1 nhưng xanh ở DB2
  • Filter Apply to "Only This Worksheet" → click filter không ảnh hưởng charts khác
🚀 Tips để Dashboard trông "thật" hơn
  • Thêm Executive Summary text box đầu mỗi dashboard: 2–3 câu key insight
  • Dùng Image object để thêm logo hoặc banner màu ở header
  • Thêm Navigation buttons (DB1 → DB2 → DB3 → DB4) bằng Button object
  • Dùng Device Preview để kiểm tra layout trên các màn hình khác nhau
  • Publish lên Tableau Public → có link chia sẻ chuyên nghiệp
